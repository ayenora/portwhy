# Roadmap

## v0.1.0

- `portwhy <port>` on macOS
- Friendly formatted output
- Basic error handling
- Clear message when no process is found

## v0.2.0

- `--json` output
- Linux support
- Parser tests with captured fixtures

## v0.3.0

- Working directory detection
- Full command-line details
- Basic framework detection

## v0.4.0

- `portwhy kill <port>` with confirmation prompt
- `--yes` flag for non-interactive use

## Later

- Windows support
- Docker awareness
- Watch mode (`portwhy watch <port>`)
- Richer framework explanations
